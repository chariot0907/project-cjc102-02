## 一、專題背景

### 凌晨三點的幽靈：阿哲的代碼長征

凌晨三點，整座城市正陷入最深沉的夢境，但阿哲的臥室卻被三台螢幕的冷光映照得如同白晝。

身為頂尖遊戲公司的雲端工程師，阿哲對安靜有著近乎病態的警覺。突然，一聲尖銳的警報聲撕裂了沉靜，像是一把手術刀，精準地劃開了他的神經。

**「伺服器全線崩潰。」**

他屏住呼吸，手指在鍵盤上瘋狂敲擊。終端機螢幕上，無數紅色的 CrashLoopBackOff 訊息像毒瘤般瘋狂增殖，那是開發者的噩夢。原本井然有序的 Pod 接連倒下，背後的流量卻像決堤的洪水，正無情地衝擊著最後一道防線。

「該死……到底在哪裡？」

阿哲的眼神布滿血絲。他像是一個在廢墟中翻找生還者的搜救員，一行行地過濾著 YAML 配置、翻找 Events、透視 Logs。但錯誤資訊支離破碎，隱藏在萬千行代碼的迷霧中。

他彷彿聽見了城市另一端，數百萬名玩家憤怒的嘶吼與焦急的等待。汗水順著他的鬢角流下，滴在冰冷的鍵盤上。時間不是在流逝，而是在**凌遲**。

就在意志即將崩塌的那一刻，世界突然晃動了一下。

---

**甦醒與救贖**

一道溫和的晨光穿透百葉窗，灑在他的指尖。

阿哲猛地驚醒，胸口劇烈起伏，手心還殘留著幻覺中的冰冷汗水。他轉頭看向桌面，螢幕安靜地顯示著系統監控圖表——一切綠燈，平穩如鏡。

「啊……原來是夢。」

他長舒了一口氣，靠在椅背上，緊繃的肩膀終於鬆弛下來。他自嘲地笑了笑，這才想起，那些在代碼廢墟中徒手挖掘的日子早已成為過去。

自從公司導入了 **K8sGPT**，那些瑣碎且令人崩潰的診斷過程，已經被精準的 AI 洞察所取代。它不再只是工具，而是一個能與 Kubernetes 靈魂溝通的智慧夥伴，在混亂發生前就指明路徑。

這場夢讓他明白，在瞬息萬變的雲端世界裡，支撐起整個平台的不再是工程師的孤軍奮戰，也不是虛無縹緲的運氣。

「謝了，老夥伴。」

他對著螢幕輕聲說道。螢幕一隅，K8sGPT 的圖示閃爍著微光，彷彿在回應這份深夜裡的默契。

## 二、專題目標

建立 **AWS EKS** 作為主要 Kubernetes Cluster 環境，搭配 **GCP GKE** 服務作為備援的混和雲架構，確保系統時刻保持啟用狀態，並透過 **K8sGPT** 提供的分析功能，協助維運工程師聚焦問題，減少 Log 紀錄的排查時間成本以及降低人為造成失誤。

## 四、痛點與解決方案

| 痛點 | 說明 |
| --- | --- |
| **問題一** | 系統事件與錯誤訊息數量龐大，難以快速判斷真正根因 |
| **問題二** | 問題排查需高度仰賴工程師個人經驗，知識難以複製 |
| **問題三** | Kubernetes 僅提供狀態與事件資訊，缺乏具體修復建議 |
| **問題四** | 在 AWS EKS 環境中，問題往往同時涉及 Kubernetes 與雲端平台設定，排錯成本高昂 |
